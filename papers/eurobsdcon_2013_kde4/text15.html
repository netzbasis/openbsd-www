KDE4 on OpenBSD: Nepomuk

  • OpenLink Virtuoso is actual storage backend

  • Soprano provides so-called high-level “models”

  • Strigi have been used by Nepomuk to extract metadata

    • and Strigi is very buggy itself
      • Nepomuk code contains auto-restarting, ever
    • so latest KDE versions started to use own-developed libextractor instead
